# プロジェクト固有のルール

## 1. 執筆・コンテンツ生成時の追加ルール

*   **メモ（ソース）からのコンテンツ生成**:
    *   ユーザーが執筆の元ネタとなるメモをチャットに提供した際、その内容を論理的に補完・肉付けしてAstro用のMarkdownページを作成する。
    *   内容のボリュームやテーマの区切りに応じて、適切な粒度でページを分割（例: 「1-1. 〇〇」「1-2. 〇〇」など）してよい。
*   **文体とトーン＆マナーの徹底**:
    *   正しく分かりやすい日本語を用い、文体は「常体（だ・である調）」で統一する。
    *   AIにありがちな誇張表現（「画期的な」「絶対に」「世界最高の」など）は使用しない。
    *   無理に不自然な比喩表現（例：教授と生徒の対話調や強引な例え話）を本文に含めず、補足情報や興味深い関連エピソードはコラム（`<div class="note-box">`）に配置する。
*   **ビジュアル解説の追加（SVGの活用）**:
    *   概念を直感的に理解しやすくするため、SVG形式の図を記事内に積極的に配置する。
    *   SVGはMarkdown内にインラインで直接記述し、ライトモードとダークモードの切り替えに対応できるよう、線の色や塗りつぶしの色にはCSSカスタムプロパティ（例: `var(--text-color)`, `var(--accent-color)`, `var(--border-color)`）を使用する。
    *   MarkdownパーサーによるHTMLブロック解釈エラー（図が正しくレンダリングされず、コードとして露出する問題）を防ぐため、インラインで直接記述するSVGタグの内部（`<svg>`〜`</svg>`）には、**絶対に空行を配置しない**こと。
*   **Markdownの太字記法における日本語文字との境界処理**:
    *   AstroやMarkdownパーサー（MDX等）が太字を正しく認識できず、星印（`**`）がそのまま露出する問題を防ぐため、太字（`**`）の「外側」に半角スペースを挿入して、日本語文字と直接密着させないようにすること。
    *   （例: `日本語**太字**日本語` は不可。 `日本語 **太字** 日本語` のように書く）
